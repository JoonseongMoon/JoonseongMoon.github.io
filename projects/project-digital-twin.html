Spring-Mass-Damper System: Physics-Based vs. Statistical Modeling

Comparing two fundamentally different approaches to predictive modeling — physics-based calibration and ARMAX statistical modeling — using a spring-mass-damper system with noisy synthetic data.

Course 12-831 Digital Twin
Year 2026
Role Graduate Coursework
Tools Python, SciPy, statsmodels
Overview

Problem Statement

In engineering systems, predictive models are essential for monitoring, control, and diagnostics. This assignment explores two fundamentally different approaches: a physics-based model grounded in first principles, and a statistical ARMAX model that learns patterns directly from data. Using a spring-mass-damper system as a test case, we examine how both approaches perform when calibrated on the same noisy measurements.

The spring-mass-damper system is governed by a second-order ordinary differential equation:

m · d²x/dt² + c · dx/dt + k · x = F(t)

Where m is the mass, c is the damping coefficient, k is the spring constant, and F(t) is the external forcing function. The true parameters used for data generation were m = 1.0, c = 0.5, k = 2.0.

Part 1

Synthetic Data Generation

Training and testing datasets were generated by solving the ODE using scipy.integrate.solve_ivp. The training data used a sine forcing function F(t) = sin(t), while the testing data used a different forcing function F(t) = 2sin(2t) to evaluate generalization. Gaussian noise (σ = 0.1) was added to simulate realistic measurement uncertainty.

data_generation.py 
# Physics-based model for spring-mass-damper system
def spring_mass_damper(t, y, m, c, k, force_func):
    x, v = y
    dxdt = v
    dvdt = (force_func(t) - c * v - k * x) / m
    return [dxdt, dvdt]

# Generate training data with sine forcing
result_train = solve_ivp(
    spring_mass_damper, t_span, y0, t_eval=t_eval,
    args=(true_mass, true_damping, true_stiffness, sine_force)
)

# Add Gaussian noise to simulate measurement noise
noisy_position = position_train + np.random.normal(0, 0.1, size=position_train.shape)
Part 2

Physics-Based Model Calibration

The physics-based approach uses a discretized Euler method to simulate the spring-mass-damper system, then calibrates the parameters (m, c, k) by minimizing the mean squared error between predicted and observed positions. Parameter estimation was performed using scipy.optimize.minimize with the L-BFGS-B method and positivity constraints.

physics_model.py 
def discretized_spring_mass_damper(params, times, forces, initial_conditions):
    m, c, k = params
    dt = times[1] - times[0]
    positions = np.zeros(len(times))
    velocities = np.zeros(len(times))
    positions[0], velocities[0] = initial_conditions

    for i in range(1, len(times)):
        # Semi-implicit Euler integration
        velocities[i] = velocities[i-1] + (forces[i-1] - c*velocities[i-1] - k*positions[i-1]) / m * dt
        positions[i] = positions[i-1] + velocities[i] * dt
    return positions

# Optimize parameters using L-BFGS-B
physics_result = minimize(
    objective_function, x0=[1.5, 0.3, 1.5],
    args=(times, positions, forces, y0),
    bounds=[(1e-6, None)]*3, method="L-BFGS-B"
)

The optimizer successfully recovered the true system parameters from noisy data, demonstrating the strength of physics-informed approaches when the underlying model structure is known.

Part 3

ARMAX Statistical Model

The ARMAX (AutoRegressive Moving Average with eXogenous inputs) model takes a purely data-driven approach, using SARIMAX from statsmodels with 5 autoregressive lags and the external force as an exogenous input. Unlike the physics-based model, ARMAX does not require knowledge of the underlying differential equation — it learns temporal patterns directly from the data.

armax_model.py 
# Fit ARMAX model with 5 AR lags and force as exogenous input
armax_model = SARIMAX(
    train_data['position'],
    exog=train_data['force'],
    order=(5, 0, 0),
    enforce_stationarity=False
)
armax_result = armax_model.fit(disp=False)

# Generate out-of-sample predictions for test data
armax_test_predictions = armax_result.apply(
    endog=test_data['position'],
    exog=test_data['force']
).fittedvalues

The AR coefficients can be loosely related back to the physical system: their sum indicates system stability, and the force coefficient reflects the mass-normalized response magnitude.

Results

Model Comparison

Key Findings

  • Physics-based model accurately recovered the true parameters (m, c, k) from noisy data
  • ARMAX model achieved strong one-step-ahead predictions on training data
  • Physics-based model generalized better to the unseen test forcing function (2sin(2t))
  • ARMAX performance degraded on test data due to distribution shift in the forcing input
  • Physics-based models leverage structural knowledge for better extrapolation; statistical models excel at interpolation

The comparison highlights a fundamental trade-off in predictive modeling: physics-based models require domain knowledge but generalize well to new conditions, while statistical models are flexible and easy to calibrate but may struggle when operating conditions change — a critical consideration for digital twin applications in building science.
